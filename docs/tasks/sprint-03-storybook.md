# Sprint 3 — Storybook Setup

Initialize Storybook 10, configure the preview with global styles, create a branded theme, and verify that all addons work with the SCSS foundation from Sprint 2.

**Estimated total: ~4 hours**

**Prerequisite:** Sprint 2 (SCSS foundation in place)

---

## Task 3.1 — Initialize Storybook 10

**Time estimate:** 1 hour

**Description:**
Run the Storybook 10 CLI to scaffold the `.storybook/` directory and install its dependencies. Storybook 10 is ESM-only and uses feature flags during init.

```bash
pnpm dlx storybook@latest init --builder vite --features docs test a11y
```

This will automatically install:

- `storybook` (core CLI)
- `@storybook/react-vite` (framework)
- `@storybook/addon-essentials` (docs, controls, actions, viewport, backgrounds)
- `@storybook/addon-a11y` (accessibility panel)
- `@storybook/addon-vitest` (component testing via Vitest)
- `@storybook/test` (test utilities)
- `@storybook/blocks` (doc blocks)

After init, verify that `.storybook/main.ts` is valid ESM and the addons array is correct. Remove any example stories generated by the CLI — we'll create our own in Sprint 4.

## Task 3.2 — Configure Preview & Global Styles

**Time estimate:** 1 hour

**Description:**
Configure `.storybook/preview.ts` to load the global SCSS styles (base reset + globals) and set default parameters for all stories.

- Import `../src/styles/main.scss` so the reset and base styles apply to every story
- Set default `parameters.layout` to `'centered'` for component stories
- Configure `parameters.controls` with expanded argTypes and sorted options
- Enable autodocs globally via `tags: ['autodocs']` in the preview config
- Set up the a11y addon default config in parameters

After configuring, run `pnpm dev` (Storybook dev server) to verify it starts without errors and the global styles are loading.

## Task 3.3 — Create Custom Storybook Theme

**Time estimate:** 1.5 hours

**Description:**
Create `.storybook/theme.ts` to brand the Storybook UI with Carleton's visual identity. Apply the theme in `preview.ts` and `manager.ts`.

- Use `@storybook/theming` `create()` function
- Set `brandTitle` to "RDS 2.0 — Carleton University"
- Set `brandUrl`, `brandImage` (Carleton logo if available)
- Apply Carleton colors: primary red (`#bf112b`) for accents, navy (`#0d3d6b`) for the sidebar
- Set typography to match the design system's font family
- Create `.storybook/manager.ts` to apply the theme to the Storybook manager UI

## Task 3.4 — Verify Addons & Vitest Integration

**Time estimate:** 30 minutes

**Description:**
Verify that all Storybook addons and the Vitest integration are working before moving to component development.

- Confirm the **Docs** addon generates an empty docs page (no components yet, just verify it loads)
